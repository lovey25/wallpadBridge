# RS485 통신 규약 명세서

## 1. 물리/링크 계층 설정

| 항목             | 값              |
| ---------------- | --------------- |
| Baud Rate        | 9600            |
| Data Bits        | 8               |
| Parity           | None            |
| Stop Bits        | 1               |
| RX Frame Timeout | 10 ms           |
| TX Interval      | 50 ms           |
| ACK Timeout      | 50 ms           |
| ACK Retry        | 1               |
| Prefix           | `F7`            |
| Suffix           | `EE`            |
| Checksum         | XOR (Checksum8) |

### Checksum 알고리즘

```cpp
crc = 0xF7
for byte in payload:
    crc ^= byte
return crc
```

---

## 2. 공통 프레임 구조

```.
[Prefix] [Length] [Device] [Command] [Data...] [Checksum] [Suffix]
```

| 필드     | 설명                    |
| -------- | ----------------------- |
| Prefix   | 프레임 시작 (`F7`)      |
| Length   | Prefix 제외 데이터 길이 |
| Device   | 장치 타입 + 주소        |
| Command  | 동작 코드               |
| Data     | 상태값 또는 설정값      |
| Checksum | XOR 결과                |
| Suffix   | 프레임 종료 (`EE`)      |

> **표기 규칙**
>
> - 본 문서의 장치별 예시는 이해를 위해 주로 `Prefix~Data`까지만 표기합니다.
> - 실제 송수신 프레임은 항상 마지막에 `Checksum`과 `Suffix(EE)`를 포함해야 합니다.

## 4. 장치별 프로토콜

### 4.1 조명 (Binary Light)

#### 상태 조회

```.
REQ: F7 0B 01 19 01 40 AA 00 00
RES: F7 0B 01 19 04 40 AA 00 SS
```

| 값      | 의미      |
| ------- | --------- |
| `AA`    | 조명 주소 |
| `SS=01` | ON        |
| `SS=02` | OFF       |

#### 제어

```.
ON  REQ: F7 0B 01 19 02 40 AA 01 00
ON  RES: F7 0B 01 19 04 40 AA 01 01

OFF REQ: F7 0B 01 19 02 40 AA 02 00
OFF RES: F7 0B 01 19 04 40 AA 02 02
```

- 조명 제어 프레임의 마지막 `00`은 예약(패딩) 바이트로, 명령 길이(`0B`)에 포함됩니다.

---

### 4.2 환기장치 (Fan)

#### 상태 조회

```.
REQ: F7 0B 01 2B 01 40 11 00 00
RES: F7 0C 01 2B 04 40 11 00 PS
```

| 필드 | 값   | 의미 |
| ---- | ---- | ---- |
| 전원 | `02` | OFF  |
| 전원 | `01` | ON   |
| 풍량 | `01` | 약   |
| 풍량 | `03` | 중   |
| 풍량 | `07` | 강   |

#### 전원 제어

```.
OFF REQ: F7 0B 01 2B 02 40 11 02 00
ON  REQ: F7 0B 01 2B 02 42 11 01 00
```

#### 풍량 제어

```.
LOW  : F7 0B 01 2B 02 42 11 01 00
MED  : F7 0B 01 2B 02 42 11 03 00
HIGH : F7 0B 01 2B 02 42 11 07 00
```

---

### 4.3 도어락

도어락(Device `1E`)은 SubCommand 바이트(`Data[0]`, 인덱스 5)에 따라
**제어 응답(`43`)** 과 **현관문 상태 broadcast(`40`)** 가 분리되어 있다.

#### 도어락 열림 제어

```.
REQ: F7 0E 01 1E 02 43 11 04 00 04 FF FF
RES: F7 0C 01 1E 04 43 11 04 00 04
```

- SubCommand `43` : 도어락 제어/응답 채널.
- 응답 데이터 첫 바이트 `04` 가 "열림 명령 수락" 의미.

#### 현관문 상태 (door sensor broadcast)

월패드는 현관문 자체의 열림/닫힘 상태를 주기적으로 broadcast 한다.

```.
RES (닫힘): F7 0C 01 1E 04 40 11 00 02 03
RES (열림): F7 0C 01 1E 04 40 11 00 02 04
```

| 필드          | 값             | 의미                |
| ------------- | -------------- | ------------------- |
| SubCommand    | `40`           | 상태 broadcast 채널 |
| Address       | `11`           | 현관문 주소         |
| Data[0..1]    | `00 02`        | 고정                |
| Data[2] (ST)  | `03`           | 닫힘 (CLOSED)       |
| Data[2] (ST)  | `04`           | 열림 (OPEN)         |

> 본 채널은 요청 없이 월패드가 자발적으로 송출하므로 별도의 폴링은 불필요하다.

---

### 4.4 난방 (Climate)

#### 상태 조회

```.
REQ: F7 0B 01 18 01 45 RR 00 00
RES: F7 0D 01 18 04 45 RR 00 SS CC TT
```

| 필드    | 의미      |
| ------- | --------- |
| `RR`    | 룸 주소   |
| `SS=01` | 난방      |
| `SS=04` | OFF       |
| `SS=07` | 외출      |
| `CC`    | 현재 온도 |
| `TT`    | 설정 온도 |

#### 전원/모드 제어

```.
HEAT REQ: F7 0B 01 18 02 46 RR 01 00
OFF  REQ: F7 0B 01 18 02 46 RR 04 00
AWAY REQ: F7 0B 01 18 02 46 RR 07 00
```

#### 온도 설정

```.
REQ: F7 0B 01 18 02 45 RR TT 00
RES: F7 0D 01 18 04 45 RR TT 01 CC TT
```

| Fan Medium | `03`           | 중          |
| Fan High   | `07`           | 강          |

---

## 6. 설계 특징

- 모든 프레임은 **Prefix(F7) / Suffix(EE)** 구조 사용
- **XOR Checksum** 기반 단순 무결성 검증
- **요청 → ACK 응답 구조**
- 장치 타입별 **Command 코드 분리**
- 온도/풍량 등은 **단일 바이트 파라미터 방식**
